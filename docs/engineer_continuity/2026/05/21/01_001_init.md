I created this repo in response to Abdul Malik's observation that the tools in `state_collapser` should have immediate application to the original graph ML message passing problem that Abdul was tackling when he introduced the ideas for simplex search that lead directly to `state_collapser`. The present `HGraphML` repo is an attempt to realize Abdul's observation.

My original idea was to have a bunch of Codex-assisted design meetings, to really pin down this `HGraphML` package in a bunch of blueprint and gameplan documentation, and then just let Codex go. But as `state_collapser` nears readiness for initial public release, I realize that it makew more sense to build something much more lightweight that calls `state_collapser` directly, and appropriates the `state_collpaser` tools for graph ML purposes. That means much less design work.

This `md` continuity report marks the creation of the repo for these purposes.
